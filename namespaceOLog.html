◆ closeLogFile()

void OLog::closeLogFile ( )

Closes the currently opened log file.

This function closes the log file if it is open. After calling this, logging to the file will not be possible until openLogFile() is called again.

◆ log()

void OLog::log ( LogLevel level,
const std::string & message,
int time = -1 )

Logs a message with a given severity level (thread-safe).

Logs messages to the file and optionally to stdout, based on the flags set. If logging a CRITICAL error and the OL_QUIT_ON_CRITICAL flag is set, the program will terminate with the specified exit code.

Parameters
levelThe severity level of the log message.
messageThe message to be logged.
time(Optional) A timestamp for the log entry. If -1, the time is not logged.
Exceptions
std::runtime_errorIf the log file is not open.

◆ openLogFile()

void OLog::openLogFile ( const std::string & file_name,
int _flags = 0 )

Opens a log file for writing.

This function attempts to open the specified log file. If successful, it appends log messages to the file. If the file is already open, an exception is thrown. A new session header is written at the beginning.

Parameters
file_nameThe name of the log file.
_flags(Optional) Flags controlling logging behavior (e.g., OL_LOG_TO_STDOUT).
Exceptions
std::logic_errorIf the log file is already open. The user must close the existing log file before opening a new one.
std::ios_base::failureIf the log file cannot be opened due to file system errors (e.g., missing permissions, invalid file path).
